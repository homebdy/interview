격리 수준은 여러 트랜잭션이 동시에 처리될 경우 특정 트랜잭션이 다른 트랜잭션에서 변경하거나 조회하고 있는 데이터를 볼 수 있게 허용할지 여부를 결정하는 것을 의미합니다.

격리수준은 크게 4가지 수준으로 나뉩니다.

먼저, 가장 엄격한 격리수준인 Serializable이 있습니다.
이 수준에서는 여러 트랜잭션이 동일한 레코드에 동시에 접근할 수 없도록 설정하여 데이터 부정합 문제가 발생하지 않도록 합니다.

2번째는 Repeatable read 입니다.
트랜잭션이 시작되기 전에 커밋된 내용에 대해서만 조회할 수 있는 격리 수준입니다.
트랜잭션이 완료될 때까지 select 문장이 사용하는 모든 데이터에 공유락을 걸어 다른 사용자들이 해당 트랜잭션 영역에 해당되는 데이터를 수정할 수 없도록 설정합니다.
이 수준에서 Non-repeatable read 부정합 문제는 발생하지 않지만, 팬텀 리드 문제가 발생합니다.

다음은 Read committed로, 트랜잭션의 변경 내용이 커밋되어야만 다른 트랜잭션에서 조회할 수 있는 수준입니다.
이 수준에서는 Non-repeatable read 부정합 문제와 팬텀 리드 문제가 모두 발생할 수 있습니다.

마지막은 Read Uncommitted로, 트랜잭션 처리중이거나 아직 커밋되지 않은 데이터를 다른 트랜잭션이 읽는 것을 허용하는 수준을 의미합니다.
데이터를 조회할 때, 해당 데이터에 락을 걸지 않기 때문에 Dirty Read 문제가 발생합니다.
이 수준은 데이터베이스의 일관성을 유지하는 것이 불가능하므로 잘 사용하지 않는 수준입니다.
